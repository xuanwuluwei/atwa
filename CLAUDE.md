## Package Management
- 唯一依赖声明：`pyproject.toml`，禁止用 `pip install` 绕过它直接装包
- 安装/重装：`pip install -e ".[dev]"`（开发）、`pip install -e ".[dev,phase2]"`（加 Phase 2）
- `aiosqlite` 是运行时依赖（SQLAlchemy async + SQLite 驱动），不是间接依赖
- 新增依赖必须声明在 `pyproject.toml` 对应的 `[dependencies]` 或 `[optional-dependencies]` 中

## Commands
- Python test: `python -m pytest`
- Python type check: `python -m mypy <path>` (逐模块检查，非整项目)
- Python lint: `ruff check .`
- DB migration: `alembic upgrade head` / `alembic revision --autogenerate`
- 前端暂未创建，无 TS 命令

## Patterns
- 配置加载顺序：`default.toml` → `<env>.toml` deep merge → `ATWA_OVERRIDE_*` env var，见 `config/loader.py`
- 所有运行时路径收敛在 `~/.atwa/<env>/`，通过 `config/paths.py` 获取，禁止硬编码
- 时间戳统一用 Unix 毫秒整数 (INTEGER)，不用 DATETIME，见 `db/models.py`
- ORM 模型在 `db/models.py`，migration DDL 在 `migrations/versions/`，两侧须保持一致
- 测试 conftest 自动设 `ATWA_ENV=test` 并清理 override 环境变量
- 接口定义优先用 TypedDict / Pydantic，模块间不导入内部实现

# Hard Rules

## Never
- 不能删除或修改已有测试用例
- 不能引入新的第三方依赖，除非被明确要求
- 代码中禁止出现魔法数字，所有阈值和可变参数必须从 `load_config()` 读取
- 不能混合不相关模块的改动
- 不能只改 ORM 模型不改 migration，或反之
- 所有文件的生成、修改、删除等写操作必须在项目目录内完成

## Must
- 生成的代码或者文档单文件不能超过 500 行
- 所有新增 public 函数必须有类型标注
- 涉及 tmux 交互的函数必须优雅处理「tmux 未运行」的情况（返回空，不抛异常）
- 数据库写入必须包含 try/except，失败记录日志但不中断采集流程
- 新功能必须有测试，覆盖 happy path 和至少一个 edge case
- 每个模块按「接口定义 → 测试桩 → 实现」顺序开发

## Before Finishing
- 建议本地跑 `ruff check <path>` + `mypy <path>` + `pytest`，作为快速反馈（PostToolUse hook 会自动触发）
- CI (GitHub Actions) 会兜底检查：push/PR 到 main 时自动跑 ruff + mypy + pytest
- 前端创建后，CI 会补跑 tsc --noEmit + eslint


## 完成任何代码改动后，必须：
1. 更新对应的 Handoff Note Dev log，记录改了什么、为什么
2. 如果改动影响了 Spec 描述的行为，同步更新 Spec
3. 没有完成以上两步，任务视为未完成
