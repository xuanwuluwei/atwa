# ATWA 设计文档 — 03 数据模型

> 单一 SQLite 文件，路径由环境配置决定（见 `02_多环境配置.md`）。
> WAL 模式开启，所有时间戳为 Unix 毫秒整数。

---

## 1. 数据库位置

| 环境 | 路径 |
|------|------|
| production | `~/.atwa/production/atwa.db` |
| development | `~/.atwa/development/atwa.db` |
| test | `~/.atwa/test/atwa.db` |

数据库初始化时执行：

```sql
PRAGMA journal_mode = WAL;
PRAGMA foreign_keys = ON;
PRAGMA busy_timeout = 30000;   -- 对应 config.database.timeout_s
```

## 2. 核心表

### 2.1 pane_sessions

记录每个被监控的 tmux pane 的完整生命周期信息。

```sql
CREATE TABLE pane_sessions (
    pane_id          TEXT PRIMARY KEY,  -- tmux pane 唯一 id，格式 %NN（稳定不变）
    tmux_session     TEXT NOT NULL,     -- tmux session 名称
    tmux_window      INTEGER NOT NULL,  -- window 索引
    tmux_pane        INTEGER NOT NULL,  -- pane 索引
    display_name     TEXT,              -- 用户自定义名称
    description      TEXT,             -- 用户自定义备注
    tags             TEXT,              -- JSON 字符串数组，如 '["backend","auth"]'
    agent_type       TEXT,              -- claude | codex | glm | other
    host_app         TEXT,              -- iterm2 | vscode | terminal | warp | other
    status           TEXT NOT NULL,     -- 见 4. Status Enum
    status_reason    TEXT,              -- 状态详情（如 prompt 文本片段）
    started_at       INTEGER,           -- session 开始时间（ms）
    ended_at         INTEGER,           -- session 结束时间（ms），NULL 表示进行中
    last_output_at   INTEGER,           -- 最近一次有输出的时间（ms）
    token_input      INTEGER DEFAULT 0,
    token_output     INTEGER DEFAULT 0,
    cost_usd         REAL DEFAULT 0,
    created_at       INTEGER NOT NULL,
    updated_at       INTEGER NOT NULL
);
```

### 2.2 tool_events

记录每次工具调用的详细信息，用于工具调用时间线和模式分析。

```sql
CREATE TABLE tool_events (
    id             INTEGER PRIMARY KEY AUTOINCREMENT,
    pane_id        TEXT NOT NULL REFERENCES pane_sessions(pane_id) ON DELETE CASCADE,
    tool_name      TEXT NOT NULL,       -- Bash | Read | Write | WebSearch | Edit | ...
    started_at     INTEGER NOT NULL,
    ended_at       INTEGER,             -- NULL 表示仍在执行
    duration_ms    INTEGER,             -- 完成后填充
    status         TEXT,                -- running | success | error
    error_summary  TEXT,                -- 错误信息前 200 字符
    raw_snippet    TEXT                 -- 原始输出片段（前 500 字符）
);

CREATE INDEX idx_tool_events_pane_id ON tool_events(pane_id);
CREATE INDEX idx_tool_events_started_at ON tool_events(started_at DESC);
```

### 2.3 interventions

记录开发者的每次干预行为，是 Insight Engine 学习的核心信号。

```sql
CREATE TABLE interventions (
    id               INTEGER PRIMARY KEY AUTOINCREMENT,
    pane_id          TEXT NOT NULL REFERENCES pane_sessions(pane_id) ON DELETE CASCADE,
    type             TEXT NOT NULL,     -- input | correction | kill | restart | skip
    content          TEXT,              -- 发送的内容或操作描述
    context_snapshot TEXT,              -- 干预前 agent 输出的最后 500 字符
    timestamp        INTEGER NOT NULL
);

CREATE INDEX idx_interventions_pane_id ON interventions(pane_id);
```

### 2.4 attention_log

记录开发者的 tmux 焦点切换历史，用于分析认知负载。

```sql
CREATE TABLE attention_log (
    id          INTEGER PRIMARY KEY AUTOINCREMENT,
    pane_id     TEXT NOT NULL,          -- 不需要外键，pane 可能已关闭
    started_at  INTEGER NOT NULL,
    ended_at    INTEGER,                -- NULL 表示当前焦点
    duration_ms INTEGER                 -- 完成后填充
);

CREATE INDEX idx_attention_log_pane_id ON attention_log(pane_id);
```

## 3. Phase 2+ 表

### 3.1 session_summaries

Phase 2 新增，在 session 结束后由 Anthropic API 生成摘要。

```sql
CREATE TABLE session_summaries (
    pane_id           TEXT PRIMARY KEY REFERENCES pane_sessions(pane_id),
    summary_text      TEXT,             -- 300 字自然语言摘要
    key_decisions     TEXT,             -- JSON 字符串数组
    tool_sequence     TEXT,             -- JSON: [{tool, count, avg_ms}]
    intervention_count INTEGER,
    correction_count  INTEGER,
    embedding_id      TEXT,             -- 对应 session_embeddings 表的 rowid
    generated_at      INTEGER
);
```

### 3.2 skill_patterns

记录工具序列的重复模式，触发 Skill 建议的基础数据。

```sql
CREATE TABLE skill_patterns (
    id                  INTEGER PRIMARY KEY AUTOINCREMENT,
    pattern_hash        TEXT UNIQUE,     -- 归一化工具序列的哈希
    tool_sequence       TEXT NOT NULL,   -- JSON
    occurrence_count    INTEGER DEFAULT 1,
    session_ids         TEXT,            -- JSON 字符串数组
    avg_duration_ms     INTEGER,
    common_corrections  TEXT,            -- JSON：最频繁的干预文本列表
    first_seen          INTEGER,
    last_seen           INTEGER
);
```

### 3.3 skill_usage

记录每次 Skill 的使用结果，用于 Skill 健康度评估。

```sql
CREATE TABLE skill_usage (
    id          INTEGER PRIMARY KEY AUTOINCREMENT,
    skill_name  TEXT NOT NULL,
    pane_id     TEXT REFERENCES pane_sessions(pane_id),
    corrections INTEGER DEFAULT 0,
    outcome     TEXT,                    -- success | partial | failed
    timestamp   INTEGER NOT NULL
);
```

### 3.4 insights

存储 Insight Engine 生成的建议。

```sql
CREATE TABLE insights (
    id          INTEGER PRIMARY KEY AUTOINCREMENT,
    type        TEXT NOT NULL,           -- new_skill | improve_skill | deprecate_skill | workflow
    title       TEXT NOT NULL,
    body        TEXT NOT NULL,           -- Markdown 正文
    evidence    TEXT,                    -- JSON: {sessions[], counts, examples[]}
    status      TEXT DEFAULT 'pending',  -- pending | accepted | snoozed | dismissed
    snoozed_until INTEGER,               -- snooze 到期时间（ms），NULL 表示未 snooze
    created_at  INTEGER NOT NULL,
    acted_at    INTEGER                  -- 用户操作时间
);

CREATE INDEX idx_insights_status ON insights(status);
```

### 3.5 session_embeddings（sqlite-vec 虚拟表）

```sql
-- 需要加载 sqlite-vec 扩展后创建
CREATE VIRTUAL TABLE session_embeddings USING vec0(
    embedding float[384]                -- fastembed all-MiniLM-L6-v2 维度
);
-- rowid 与 session_summaries.embedding_id 关联
```

## 4. Status Enum

pane_sessions.status 字段的所有合法值：

| 状态码 | 标识 | 描述 | 自动流转到 |
|--------|------|------|-----------|
| `waiting_input` | 🔴 | 检测到交互提示（[y/n]、Continue?、Input:） | active（发送后） |
| `error_stopped` | 🔴 | 检测到错误/异常且进程停止 | dead / restarted |
| `cost_alert` | 🔴 | Token 或费用超过阈值 | active（确认后） |
| `stuck` | 🔴 | 检测到重复输出（循环） | active / killed |
| `retry_loop` | 🟠 | 同一操作重试 3+ 次 | active / waiting_input |
| `slow_tool` | 🟠 | 单次工具调用超时 | active |
| `high_error_rate` | 🟠 | 最近 10 次工具调用失败率 > 50% | active |
| `active` | 🟡 | 持续有输出 | — |
| `tool_executing` | 🟡 | 工具调用进行中 | active / error |
| `thinking` | 🟡 | LLM 推理中，无工具调用 | tool_executing / active |
| `waiting_tool` | 🔵 | 工具已调用，等待外部结果 | active / error |
| `idle_running` | 🔵 | 进程存活但无输出（未超过 long 阈值） | active / stuck |
| `completed` | 🟢 | 检测到明确的完成信号 | — |
| `idle_long` | 🟢 | 长时间无输出，可能已完成 | — |
| `terminated` | 🟢 | 进程正常退出（exit 0） | — |
| `crashed` | ⚫ | 异常退出（非零 exit code） | — |
| `killed` | ⚫ | 收到 SIGKILL 或 SIGTERM | — |

### 状态优先级（用于 get_attention_queue 排序）

```
🔴 waiting_input > error_stopped > cost_alert > stuck
🟠 retry_loop > slow_tool > high_error_rate
🟡 active > tool_executing > thinking
🔵 waiting_tool > idle_running
🟢 completed > idle_long > terminated
⚫ crashed > killed
```

## 5. Schema 迁移规范

使用 alembic 管理 schema 版本，迁移文件存放在 `migrations/versions/`。

```ini
# alembic.ini
[alembic]
script_location = migrations
# 数据库 URL 由运行时配置注入，不硬编码在此文件
sqlalchemy.url = 通过 env.py 动态获取
```

```python
# migrations/env.py
from config.loader import load_config
from config.paths import get_paths
import os

env_name = os.getenv("ATWA_ENV", "production")
paths = get_paths(env_name)
config.set_main_option("sqlalchemy.url", f"sqlite+aiosqlite:///{paths['db']}")
```

**迁移命令**：

```bash
# 创建新迁移
ATWA_ENV=development alembic revision --autogenerate -m "add_skill_patterns"

# 应用迁移（指定环境）
ATWA_ENV=production alembic upgrade head
ATWA_ENV=test alembic upgrade head
```
