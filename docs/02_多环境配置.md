# ATWA 设计文档 — 02 多环境配置

> 本文档定义 ATWA 在开发（development）、测试（test）、生产（production）三个环境中的完整配置规范。
> **核心约束**：所有运行时产物（数据库、日志、PTY 录制文件、临时文件）必须收敛在 `~/.atwa/<env>/` 目录下，不得散落到系统其他位置。

---

## 1. 环境边界收敛原则

### 1.1 目录规划

所有环境的运行时产物统一存放在用户 Home 目录下，按环境隔离：

```
~/.atwa/
├── production/                 # 日常使用（默认环境）
│   ├── atwa.db                 # 主数据库
│   ├── logs/
│   │   ├── daemon.log
│   │   ├── server.log
│   │   └── pty/                # PTY 录制文件（按 pane_id 命名）
│   │       ├── pane-%1.log
│   │       └── pane-%2.log
│   └── tmp/                    # 临时文件（启动锁、pid 文件）
│       ├── daemon.pid
│       └── server.pid
│
├── development/                # 本地开发调试
│   ├── atwa.db
│   ├── logs/
│   │   ├── daemon.log
│   │   ├── server.log
│   │   └── pty/
│   └── tmp/
│
└── test/                       # 自动化测试（每次测试前清空）
    ├── atwa.db
    ├── logs/
    │   ├── daemon.log
    │   ├── server.log
    │   └── pty/
    └── tmp/
```

### 1.2 路径解析规则

代码中所有路径通过 `config.paths` 对象统一获取，禁止在代码中硬编码任何绝对路径：

```python
# config/paths.py
from pathlib import Path
import os

def get_base_dir(env: str) -> Path:
    """所有运行时文件的根目录，收敛在 ~/.atwa/<env>/"""
    return Path.home() / ".atwa" / env

def get_paths(env: str) -> dict:
    base = get_base_dir(env)
    return {
        "base":     base,
        "db":       base / "atwa.db",
        "log_dir":  base / "logs",
        "pty_dir":  base / "logs" / "pty",
        "tmp_dir":  base / "tmp",
        "daemon_log": base / "logs" / "daemon.log",
        "server_log": base / "logs" / "server.log",
        "daemon_pid": base / "tmp" / "daemon.pid",
        "server_pid": base / "tmp" / "server.pid",
    }
```

启动时自动创建所有必要目录：

```python
def ensure_dirs(env: str):
    paths = get_paths(env)
    for key in ["log_dir", "pty_dir", "tmp_dir"]:
        paths[key].mkdir(parents=True, exist_ok=True)
```

## 2. 配置文件结构

配置采用 TOML 分层覆盖机制：`default.toml` 定义所有字段的基准值，各环境文件只写需要覆盖的项。

### 2.1 default.toml（基准配置）

```toml
# config/default.toml
# 所有环境的基准值，各环境文件覆盖需要变更的项

[env]
name = "production"             # 当前环境名，由启动脚本注入

[paths]
# 所有路径相对于 ~/.atwa/<env>/，代码通过 get_paths() 获取
# 此处仅为文档说明，实际路径由 paths.py 动态计算

[daemon]
capture_interval_active_ms = 500    # 活跃 pane 的轮询间隔
capture_interval_idle_ms   = 3000   # 空闲 pane 的轮询间隔
scrollback_lines            = 200   # 每次 capture-pane 抓取的行数

[thresholds]
idle_running_s   = 30       # active → idle_running 的阈值（秒）
idle_long_s      = 300      # idle_running → idle_long 的阈值（秒）
slow_tool_s      = 180      # tool_executing → slow_tool 的阈值（秒）
stuck_similarity = 0.92     # stuck 检测的相似度阈值（0-1）
stuck_window     = 5        # stuck 检测的滑动窗口大小

[costs]
alert_usd_per_session = 1.00    # 单 session 费用告警阈值（美元）

[server]
host = "127.0.0.1"
port = 8742

[log]
level          = "INFO"
max_bytes      = 10_485_760     # 单个日志文件最大 10MB
backup_count   = 3              # 保留 3 个历史日志文件

[pty]
max_bytes      = 10_485_760     # 单个 PTY 录制文件最大 10MB
backup_count   = 3

[database]
journal_mode = "WAL"
timeout_s    = 30
pool_size    = 5

[tmux]
socket = ""                     # 空字符串表示使用默认 socket
                                # 多 server 场景填 socket 名称（-L 参数）

[insight_engine]
enabled          = true
anthropic_model  = "claude-sonnet-4-20250514"
similarity_threshold = 0.88     # 跨 session 相似度匹配阈值
min_occurrences  = 5            # 触发 new_skill 建议的最低重复次数
```

## 3. 环境加载机制

### 3.1 环境变量优先级

```
环境变量 ATWA_ENV > 启动参数 --env > 默认值 production
```

环境变量说明：

| 变量名 | 用途 | 默认值 |
|--------|------|--------|
| `ATWA_ENV` | 当前环境名（development/test/production） | `production` |
| `ATWA_CONFIG_DIR` | 配置文件目录（覆盖默认的 `config/`） | `<项目根>/config` |
| `ATWA_OVERRIDE_*` | 覆盖任意配置项，如 `ATWA_OVERRIDE_SERVER_PORT=9000` | 无 |

### 3.2 配置加载顺序

```python
# config/loader.py
import tomllib
from pathlib import Path

def load_config(env: str = None) -> dict:
    """
    加载顺序：
    1. config/default.toml（基准）
    2. config/<env>.toml（环境覆盖）
    3. 环境变量 ATWA_OVERRIDE_* （运行时覆盖，仅用于 CI/测试）
    """
    env = env or os.getenv("ATWA_ENV", "production")
    config_dir = Path(os.getenv("ATWA_CONFIG_DIR", "config"))

    # 1. 加载基准配置
    with open(config_dir / "default.toml", "rb") as f:
        cfg = tomllib.load(f)

    # 2. 合并环境配置（深度合并，非浅覆盖）
    env_file = config_dir / f"{env}.toml"
    if env_file.exists():
        with open(env_file, "rb") as f:
            env_cfg = tomllib.load(f)
        cfg = deep_merge(cfg, env_cfg)

    # 3. 环境变量覆盖（ATWA_OVERRIDE_SECTION_KEY=value）
    for key, val in os.environ.items():
        if key.startswith("ATWA_OVERRIDE_"):
            parts = key[14:].lower().split("_", 1)
            if len(parts) == 2:
                section, field = parts
                if section in cfg:
                    cfg[section][field] = _cast(val, cfg[section].get(field))

    cfg["env"]["name"] = env
    return cfg
```

## 5. 日志规范

### 5.1 日志文件管理

所有日志文件使用 Python `logging.handlers.RotatingFileHandler`：

```python
# daemon/main.py 和 server/main.py 中的日志初始化
import logging
from logging.handlers import RotatingFileHandler
from config.loader import load_config
from config.paths import get_paths

def setup_logging(env: str):
    cfg = load_config(env)
    paths = get_paths(env)

    handler = RotatingFileHandler(
        paths["daemon_log"],                    # 收敛在 ~/.atwa/<env>/logs/
        maxBytes=cfg["log"]["max_bytes"],        # 默认 10MB
        backupCount=cfg["log"]["backup_count"],  # 默认保留 3 份
        encoding="utf-8",
    )
    logging.basicConfig(
        level=getattr(logging, cfg["log"]["level"]),
        handlers=[handler],
        format="%(asctime)s [%(levelname)s] %(name)s: %(message)s",
    )
```

### 5.2 PTY 录制文件管理

```python
# daemon/capture.py
def get_pty_log_path(pane_id: str, env: str) -> Path:
    """
    PTY 录制文件路径，全部收敛在 ~/.atwa/<env>/logs/pty/ 下
    文件名：pane-<pane_id 去掉 %>-<timestamp>.log
    示例：pane-23-1714000000.log
    """
    paths = get_paths(env)
    clean_id = pane_id.lstrip("%")
    return paths["pty_dir"] / f"pane-{clean_id}.log"
```

PTY 文件同样使用 RotatingFileHandler 逻辑，超过 `pty.max_bytes` 后自动轮转。

### 5.3 测试环境日志清理

测试框架的 `conftest.py` 在每次测试套件开始前清空测试环境的日志和数据库：

```python
@pytest.fixture(scope="session", autouse=True)
def clean_test_env():
    base = Path.home() / ".atwa" / "test"
    # 清空数据库
    db_path = base / "atwa.db"
    if db_path.exists():
        db_path.unlink()
    # 清空日志（保留目录）
    for f in (base / "logs").rglob("*.log"):
        f.unlink(missing_ok=True)
    ensure_dirs("test")
    yield
```

## 6. 端口与进程冲突防护

### 6.1 启动检查

Daemon 和 Server 启动时写入 pid 文件，防止重复启动：

```python
# 路径：~/.atwa/<env>/tmp/daemon.pid
def check_not_running(pid_file: Path) -> None:
    if pid_file.exists():
        pid = int(pid_file.read_text().strip())
        try:
            os.kill(pid, 0)     # 检查进程是否存活
            raise RuntimeError(f"ATWA daemon already running (pid={pid})")
        except ProcessLookupError:
            pid_file.unlink()   # 残留的 pid 文件，清除后继续

def write_pid(pid_file: Path) -> None:
    pid_file.write_text(str(os.getpid()))

def cleanup_pid(pid_file: Path) -> None:
    pid_file.unlink(missing_ok=True)
```
