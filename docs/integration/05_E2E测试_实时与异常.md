## 4.8 状态实时更新

**文件**：`tests/e2e/test_realtime.spec.ts`

---

#### TC-E2E-RT-001：WebSocket 断线自动重连

**目的**：验证 WebSocket 连接断开后，UI 自动重连并恢复实时更新。

**测试步骤**：
```typescript
test('WebSocket 断线重连', async ({ page }) => {
  await dashboard.goto();

  await stopServer();

  // UI 显示断线状态
  await expect(page.locator('[data-testid="ws-status"]')).toHaveClass(/disconnected/, { timeout: 3000 });

  await startServer();

  // 5 秒内自动重连
  await expect(page.locator('[data-testid="ws-status"]')).toHaveClass(/connected/, { timeout: 5000 });

  // 重连后状态更新正常
  await triggerStatusChange('%1', 'waiting_input');
  const card = await dashboard.getCardByPaneId('%1');
  await expect(card.locator('[data-testid="status-badge"]')).toContainText('waiting_input', { timeout: 2000 });
});
```

**失败排查**：
- 若重连后状态不更新：检查前端 WebSocket hook 是否在重连后重新发起 `/api/sessions` 请求以补全断线期间的状态变更

---

#### TC-E2E-RT-002：状态变更 1 秒内反映到 UI

**目的**：端到端验证从 daemon 检测到状态变更到 UI 更新的延迟 <= 1 秒。

**测试步骤**：
```typescript
test('状态更新延迟 <= 1s', async ({ page }) => {
  await dashboard.goto();

  const changeTime = Date.now();
  await triggerStatusChange('%1', 'waiting_input');

  const card = await dashboard.getCardByPaneId('%1');
  await expect(card.locator('[data-testid="status-badge"]'))
    .toContainText('waiting_input', { timeout: 1000 });

  expect(Date.now() - changeTime).toBeLessThan(1000);
});
```

**失败排查**：
- 若偶发超时：检查 CI 环境 CPU 是否被限制；将 timeout 放宽到 2000ms 并记录实际延迟用于分析
- 若持续超时：检查 `CAPTURE_INTERVAL_ACTIVE_MS` 是否被覆盖为更大值；检查 WebSocket 广播是否被 DB 写入阻塞

---

#### TC-E2E-RT-003：计时器持续更新

**目的**：验证 running 状态的 session card 上的计时器每秒自动更新。

**测试步骤**：
```typescript
test('计时器持续更新', async ({ page }) => {
  const card = await dashboard.getCardByPaneId('%1');  // active 状态

  const time1 = await card.locator('[data-testid="elapsed-timer"]').textContent();
  await page.waitForTimeout(2000);
  const time2 = await card.locator('[data-testid="elapsed-timer"]').textContent();

  expect(time1).not.toBe(time2);
});
```

## 4.10 异常场景与降级处理

**文件**：`tests/e2e/test_edge_cases.spec.ts`

---

#### TC-E2E-EDGE-001：tmux 不可用时的 UI 处理

**前置条件**：停止测试 tmux server。

**预期结果**：
- Dashboard 仍可加载，不白屏
- 显示 banner：`"tmux server 未检测到，请启动 tmux 后刷新"`
- session 列表显示空状态（非报错页面）
- 无 JS 错误

---

#### TC-E2E-EDGE-002：20 个 session 渲染性能

**前置条件**：预置 20 个 pane sessions 数据。

**测试步骤**：
```typescript
test('20个session渲染性能', async ({ page }) => {
  const startTime = Date.now();
  await dashboard.goto();
  await page.waitForSelector('[data-testid="session-card"]:nth-child(20)');
  const renderTime = Date.now() - startTime;

  expect(renderTime).toBeLessThan(2000);

  // 验证滚动不卡顿
  await page.evaluate(() => window.scrollTo(0, document.body.scrollHeight));
  await page.evaluate(() => window.scrollTo(0, 0));
});
```

**预期结果**：
- 20 张卡片在 2 秒内全部渲染完成
- 滚动无明显卡顿

---

#### TC-E2E-EDGE-003：长文本 description 省略显示

**前置条件**：pane `%1` 的 description 为 500 字的长文本。

**预期结果**：
- 默认显示一行，末尾有省略号 `...`
- hover 或点击「展开」后显示完整文本
- 不破坏卡片整体布局

---

#### TC-E2E-EDGE-004：API 请求失败的错误提示

**目的**：验证 API 请求失败时有用户友好的提示，而非白屏或无响应。

**测试步骤**：
```typescript
test('API 请求失败提示', async ({ page }) => {
  await page.route('/api/sessions', route => route.fulfill({ status: 500 }));
  await dashboard.goto();

  await expect(page.locator('[data-testid="error-banner"]')).toBeVisible();
  await expect(page.locator('[data-testid="error-banner"]')).toContainText('加载失败');
  await expect(page.locator('[data-testid="retry-btn"]')).toBeVisible();
});
```

**预期结果**：
- 显示错误 banner 而非空白页
- 提供「重试」按钮，点击后重新发起请求
- 无 JS 未捕获异常
