## 1. 项目概览

**ATWA（AI Terminal Workflow Assistant）** 是一个开发者生产力工具，用于监控在 tmux 中运行的多个 AI agent 终端会话（Claude、Codex、GLM 等），通过 Web UI 展示可操作的状态信息，并随时间学习开发者的使用习惯，主动提出工作流改进建议。

### 核心设计原则

- **非侵入式观测** — 不替换或包装 AI CLI，只在旁路观察
- **零服务部署** — 全部以本地进程运行，单个 SQLite 文件存储
- **Harness 优先工程** — 每个阶段交付独立可测试、可被 agent 执行的单元
- **渐进式智能** — Phase 1 立即提供价值，Phase 2/3 在此基础上叠加

### 项目元信息

| 属性 | 值 |
|------|-----|
| 目标平台 | macOS（Apple Silicon / Intel） |
| 终端应用 | iTerm2 + tmux |
| 主要语言 | Python 3.11+ / TypeScript（前端） |
| 架构模式 | Daemon + FastAPI + React Web UI |

## 2. 系统架构

### 2.1 架构总图

```
tmux server（所有 session / window / pane）
        │
        │  libtmux capture-pane  +  PTY log 文件（script 命令）
        ▼
┌─────────────────────────────────────────────────────┐
│                  Daemon 进程                          │
│  SessionTracker    — pane 发现与生命周期管理           │
│  EventParser       — 工具调用/错误/提示/卡死检测       │
│  PatternRecorder   — 事件写入 SQLite                  │
│  InsightScheduler  — Phase 3 事件驱动触发器           │
└──────────────────────┬──────────────────────────────┘
                       │
                       │  SQLite（WAL 模式）+ sqlite-vec 扩展
                       │
┌──────────────────────▼──────────────────────────────┐
│                  FastAPI 进程                         │
│  REST API          — session CRUD、元数据、动作        │
│  WebSocket         — 实时状态推送到前端               │
│  ActionHandler     — tmux send-keys、AppleScript 跳转 │
└──────────────────────┬──────────────────────────────┘
                       │
┌──────────────────────▼──────────────────────────────┐
│                  React Web UI                         │
│  Dashboard         — 状态卡片、过滤器、排序            │
│  SessionDetail     — 工具调用时间线、运行时信息        │
│  InsightPanel      — 主动建议面板                     │
│  InputWidget       — 内联 send-keys + 快捷回复        │
└─────────────────────────────────────────────────────┘
```

### 2.2 进程拓扑

两个独立的 OS 进程，由 launchd plist 管理：

| 进程 | 入口文件 | 崩溃后重启 | 依赖 |
|------|----------|-----------|------|
| `atwa-daemon` | `daemon/main.py` | 是（launchd） | tmux server、SQLite |
| `atwa-server` | `server/main.py` | 是（launchd） | SQLite、daemon 写入的数据 |

> **关键设计**：Daemon 与 Server 仅通过 SQLite 通信。Server 崩溃时，Daemon 继续采集数据，数据不丢失。

## 4. 项目文件结构

所有路径均相对于项目根目录 `atwa/`，所有运行时产物（数据库、日志、PTY 录制文件）均收敛在 `~/.atwa/` 下，详见 `02_多环境配置.md`。

```
atwa/
├── daemon/
│   ├── main.py                 # 进程入口，编排各子模块，主轮询循环
│   ├── tmux_discovery.py       # PaneInfo 枚举
│   ├── capture.py              # capture-pane + PTY log 双模式采集
│   ├── event_parser.py         # ParsedEvent 解析
│   ├── session_tracker.py      # 状态机，驱动 status 转换
```

## 5. 开发阶段规划

| 阶段 | 目标 | 关键交付物 |
|------|------|-----------|
| **Phase 1** | 感知 + Dashboard | Daemon 采集、状态分类、Web UI、iTerm2 跳转、内联输入 |
| **Phase 2** | 记忆层 | SQLite 结构化记忆、sqlite-vec 语义记忆、Skill 生命周期 |
| **Phase 3** | 智能层 | 事件驱动 Insight Engine、带证据链的建议、跨 session 上下文注入 |

每个阶段的详细模块规范见对应文档（`04` ~ `06`）。
